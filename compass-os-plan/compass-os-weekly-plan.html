Read this with the SOW. The Statement of Work says what we're building and how Compass pays for it. This plan says how it gets built, week by week.
What's in this plan.
  • 16 weeks, ticket by ticket. Grouped by stream: Web app (A), Bruce on WhatsApp (B), Agent space (C).
  • What Compass does each week. Most weeks: a 30-minute Monday catch-up and a 5-minute Friday Loom.
  • Three milestone gates: M1, M2, M3. Each one carries an acceptance test, taken straight from the SOW.
  • The kick-off pack we promise in the SOW. Every account, every brand asset, every decision Compass locks in before week 1.

Who builds this

Same team the SOW describes. Three of us on the build, plus a small specialised agent set behind us for drafting and scaffolding. Agents draft, we review, then it ships.

Frontend Designer

DevOps

Tester
Tries to break the build before clients do. Walks the end-to-end flow before each milestone.

Risks and how we handle them

Same three risks the SOW names. Worth repeating here so we hold them in mind every week.

Risk 01
WhatsApp transport reliability

Bruce's WhatsApp number can drop. Outbound can be rate-limited if too aggressive. The whole capture story leans on this working.

How we handle it. WhatsApp is the transport, not the source of truth. Every message mirrors into Supabase as an append-only event log. If Bruce drops, capture pauses, records survive. Outbound is rate-limited. Heartbeat monitoring runs from Week 14. Re-pair runbook lives in the repo from Week 1.
Risk 02
Historical archive is messier than expected

2.5 years of WhatsApp + email + Drive is a lot of data, much of it unique to Compass. Ingestion may sit lower than expected on first pass.

How we handle it. Stage 1 historical scope is archive-grade only: files filed, searchable, retrievable, defensible if audited. Structured parsing is a Phase 2 SOW. The Week 11 acceptance gate is "ingested + searchable", not "100% structured".
Risk 03
Scope grows mid-build

The team will see the build and have ideas. Real client work will surface patterns the original spec didn't catch.

How we handle it. Inside-scope work gets reordered in real time without paperwork. Outside-scope work gets a one-page change note (impact, cost, knock-on) that Compass confirms in writing before anything starts. No surprises either way.
